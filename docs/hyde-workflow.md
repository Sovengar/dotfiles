## 1. Punto de Entrada: `hyprland.lua`

`~/.config/hypr/hyprland.lua` es el archivo principal que Hyprland carga. Carga mГіdulos en orden estricto:

1. **Base HyDE**: `primary_apps` вҶ’ `variables` вҶ’ `env` вҶ’ `hardware/*` вҶ’ `style.opacity` вҶ’ `layout` вҶ’ `misc` вҶ’ `gestures` вҶ’ `startup`
2. **Estilo dinГЎmico**: `style.theme` вҶҗ lee estado generado en `~/.local/state/hypr/`
3. **Animaciones/Shaders**: `style.animations` вҶҗ lee `animations.theme.conf` del state
4. **Workflows**: `selected_workflow_loader` вҶҗ lee `HYPR_WORKFLOW` del `staterc`
5. **Keybindings**: `keybindings`

Los mГіdulos Lua usan `~/.local/lib/?.lua` como path, ahГӯ vive la **API de HyDE** (`hl.config()`, `hl.on()`, `hl.exec_cmd()`, `hl.curve()`, `hl.animation()`).

## 2. Estado DinГЎmico: `~/.local/state/hypr/`

Archivos **autogenerados**, NO editar manualmente (se sobreescriben en cada `hyprctl reload`):

| Archivo | Contenido | QuiГ©n lo genera |
|---|---|---|
| `staterc` | Variables de estado: `HYDE_THEME`, `HYPR_ANIMATION`, `HYPR_WORKFLOW`, `HYPRLOCK_LAYOUT`, `HYPR_SHADER` | HyDE theme switcher |
| `colors.conf` | 4 grupos de colores wallbash (`$wallbash_pry1`..`$wallbash_4xa9`) en hex + rgba | wallbash |
| `wallbash.conf` | Variables Hyprlang derivadas del theme (`$HYDE_THEME`, `$GTK_THEME`, `$ICON_THEME`, etc.) | wallbash |
| `hyprland.theme.lua` | Tabla Lua con gaps, borders, blur, rounding, colores activo/inactivo | theme.switch.sh |
| `animations.theme.conf` | Bloques de animaciГіn Hyprlang (bezier curves + animation rules) | wallbash desde `.dcol` |
| `waybar.theme.conf` | `rounding = 10` (legacy, usado por Hyprland) | theme.switch.sh |
| `shaders.conf` | Shader activo (`$SCREEN_SHADER`) y ruta al `.frag` compilado | shaders.sh |
| `hyprlock` | Path al layout de hyprlock activo | hyprlock.sh |
| `hyprsunset` | Config de temperatura de pantalla | hyprsunset |
| `metadata.conf` | Snapshot legible del theme actual (fuentes, iconos, cursor, etc.) | metadata_generator.lua |

### 2b. Estado DinГЎmico Waybar: `~/.local/state/waybar/`

Archivos **autogenerados**, NO editar manualmente (se sobreescriben en cada cambio de layout/theme):

| Archivo | Contenido | QuiГ©n lo genera |
|---|---|---|
| `staterc` | Variables: `WAYBAR_LAYOUT_PATH`, `WAYBAR_LAYOUT_NAME`, `WAYBAR_STYLE_PATH`, `WAYBAR_SCALE` | waybar layout switcher |
| `generated/config.jsonc` | Config JSON completa de waybar (mГіdulos, grupos, layout) | theme.switch.sh |
| `generated/style.css` | CSS maestro: importa defaults, border-radius, wallbash colors, theme, user overrides | theme.switch.sh |
| `generated/theme.css` | Colores del theme activo (`bar-bg`, `main-bg`, `main-fg`, `wb-act-bg/fg`, `wb-hvr-bg/fg`) | wallbash via `waybar.dcol` |
| `generated/includes/includes.json` | Lista de mГіdulos disponibles + overrides de tamaГұo de iconos | theme.switch.sh |
| `generated/includes/border-radius.css` | Border-radius dinГЎmico por forma (leaf, pill) sincronizado con Hyprland rounding | theme.switch.sh |
| `generated/includes/global.css` | `font-family`, `font-size`, `border-radius` base | theme.switch.sh |

**JerarquГӯa de carga CSS en waybar** (orden de `style.css`):
1. `~/.local/share/waybar/styles/defaults.css` вҖ” estilos base estГЎticos
2. `includes/border-radius.css` вҖ” rounding dinГЎmico por forma
3. `includes/global.css` вҖ” fuentes y tamaГұo base
4. `~/.cache/hyde/wallbash/gtk.css` вҖ” colores wallbash (@define-color)
5. `theme.css` вҖ” colores del theme activo (sobreescribe wallbash)
6. `~/.config/waybar/user-style.css` вҖ” overrides del usuario (vacГӯo por defecto)

**Layouts**: Los layouts JSON viven en `~/.config/waybar/layouts/` (usuario) y `~/.local/share/waybar/layouts/hyprdots/` (estГЎticos). El staterc apunta al layout activo. Backups en `layouts/backup/`.

**Formato `.t2`**: Como `.dcol` pero para archivos binarios/estructurados (SVG).

## 4. Sistema de Colores Wallbash

### Paleta de 4 Grupos

Wallbash extrae **4 colores primarios** del wallpaper, cada uno con:
- **pry** (primary): color base del grupo
- **txt** (text): color de texto sobre ese primary
- **xa1..xa9** (accent 1-9): 9 tonos de accent desde oscuro вҶ’ claro

```
Grupo 1: wallbash_pry1, wallbash_txt1, wallbash_1xa1..1xa9  (fondo principal)
Grupo 2: wallbash_pry2, wallbash_txt2, wallbash_2xa1..2xa9  (accent secundario)
Grupo 3: wallbash_pry3, wallbash_txt3, wallbash_3xa1..3xa9  (highlight)
Grupo 4: wallbash_pry4, wallbash_txt4, wallbash_4xa1..4xa9  (contraste/emphasis)
```

### Flujo de generaciГіn

```
Wallpaper (imagen)
       в”Ӯ
       в–ј
  wallbash (API en ~/.local/lib/hyde/)
       в”Ӯ  Extrae 4 colores dominantes вҶ’ genera paleta 4Г—(1+1+9) = 44 colores
       в”Ӯ  Cada color disponible en: hex, rgba, rgb
       в–ј
  Plantillas .dcol/.Xcol/.t2 en ~/.local/share/hyde/wallbash/
       в”Ӯ  <wallbash_pry1> вҶ’ 292749    (reemplazo en tiempo real)
       в–ј
  Archivos finales en ~/.local/state/hypr/ y ~/.cache/hyde/wallbash/
       в”Ӯ
       в–ј
  Apps consumen estos archivos
```

### Modos wallbash

El campo `HYPR_ANIMATION` en `staterc` indica el modo actual (`theme`), que determina si se usan las plantillas `always/`, `theme/`, o ambas.

## 5. CГіmo se Aplica el Theme a Cada App

### Hyprland (compositor)
- `hyprland.lua` вҶ’ carga `style/theme.lua` вҶ’ lee `hyprland.theme.lua` del state вҶ’ aplica gaps, borders, blur, colores de borde activo/inactivo
- `colors.conf` se sourcea como variables Hyprlang (`$wallbash_pry1`, etc.)
- Shaders: `shaders.conf` вҶ’ compila `.frag` a cache вҶ’ `decoration.screen_shader`

### Kitty (terminal)
- Plantilla `kitty.dcol` вҶ’ genera `~/.config/kitty/autogenerated_theme.conf`
- `defaults_from_hyde.conf` hace `include autogenerated_theme.conf`
- Colores: foreground, background, cursor, 16 colores ANSI, tabs

### VSCode / Code Editors
- Plantilla `code.dcol` вҶ’ genera `~/.cache/hyde/wallbash/code.json`
- JSON completo con ~530 propiedades de color (editor, sidebar, terminal, etc.)
- `$CODE_THEME=Wallbash` en `wallbash.conf` indica que se usa el theme autogenerado

### GTK (apps nativas: Dolphin, Nautilus, etc.)
- Plantillas `gtk/gtk2.dcol`, `gtk3.dcol`, `gtk4.dcol` вҶ’ themes GTK
- `gtk-css.dcol` вҶ’ `~/.cache/hyde/wallbash/gtk.css` con `@define-color` wallbash
- `settings.ini` вҶ’ `gtk-theme-name=Rose-Pine`, `gtk-icon-theme-name=Tela-circle-pink`
- Variables env: `GTK_THEME=Wallbash-Gtk`

### Waybar (barra) вҖ” dos capas de estado dinГЎmico
- **Estado Hyprland** (`~/.local/state/hypr/waybar.theme.conf`): rounding value legacy (leГӯdo por Hyprland)
- **Estado Waybar** (`~/.local/state/waybar/`): toda la config dinГЎmica de waybar
  - `staterc`: `WAYBAR_LAYOUT_PATH`, `WAYBAR_LAYOUT_NAME`, `WAYBAR_STYLE_PATH`, `WAYBAR_SCALE`
  - `generated/config.jsonc`: layout de mГіdulos, grupos, posiciones
  - `generated/style.css`: CSS maestro que importa en cascada:
    1. `defaults.css` (estГЎticos)
    2. `border-radius.css` (sync con Hyprland rounding)
    3. `global.css` (fuentes/tamaГұo)
    4. `gtk.css` wallbash (@define-color)
    5. `theme.css` (colores del theme activo)
    6. `user-style.css` (overrides del usuario)
  - `generated/theme.css`: colores concretos (`bar-bg`, `main-bg/fg`, `wb-act-*`, `wb-hvr-*`)
  - `generated/includes/`: mГіdulos disponibles + icon-size overrides
- Plantilla `waybar.dcol` вҶ’ genera los CSS con colores wallbash
- Layouts: `~/.config/waybar/layouts/` (usuario) + `~/.local/share/waybar/layouts/hyprdots/` (estГЎticos)
- `user-style.css` вҶ’ overrides del usuario (vacГӯo por defecto)

### Rofi (launcher)
- Plantilla `rofi.dcol` вҶ’ `~/.config/rofi/theme.rasi`
- Variables: `main-bg`, `main-fg`, `main-br`, `main-ex`, `select-bg`, `select-fg`

### Dunst (notificaciones)
- Plantilla `dunst.dcol` вҶ’ `~/.cache/hyde/wallbash/dunst.conf`
- Script post-procesador: `dunst.sh` copia y recarga dunst
- Colores para urgencia low/normal/critical

### SwayNC (centro de notificaciones)
- Plantilla `swaync.dcol` вҶ’ `~/.config/swaync/theme.css`
- Importa `gtk.css` wallbash + define colores CSS propios

### Hyprlock (lock screen)
- Plantilla `hyprlock.dcol` вҶ’ config de lock screen
- `hyprlock_background.dcol` вҶ’ genera imagen de fondo
- Layout seleccionado via `HYPRLOCK_LAYOUT` en `staterc` (ej: "SF Pro")

### Qt Apps (Dolphin, etc.)
- Plantilla `qtct.dcol` вҶ’ `~/.cache/hyde/wallbash/qtct.conf`
- Script post-procesador: `qtct.sh` aplica colores Qt

### Kvantum (Qt theming engine)
- Plantillas `kvantum/kvantum.dcol` + `kvantum/kvconfig.dcol` вҶ’ theme SVG

### Shell / Scripts
- `shell-colors.dcol` вҶ’ variables de shell para uso en scripts
- `pywal-colors.Xcol` вҶ’ formato pywal (`~/.cache/wal/colors`) para compatibilidad

### Shaders GLSL
- `hyprshaders.dcol` вҶ’ `#define` en GLSL para uso en fragment shaders
- Shaders en `~/.local/share/hypr/shaders/*.frag` (ej: `vibrance.frag`)
- Compilados a `~/.cache/hypr/shaders/.compiled.cache.glsl`

## 6. Flujo de Arranque

### 6.1 UWSM (bootstrap)

[UWSM](https://github.com/Vladimir-csp/uwsm) arranca Hyprland como unidad systemd user, antes de que `hyprland.lua` se ejecute:

```
login вҶ’ PAM вҶ’ uwsm sets env вҶ’ shell init (sobreescrito por uwsm) вҶ’ hyprland starts
```

Implicaciones:
- Variables de entorno las fija UWSM (PAM вҶ’ systemd), no el shell init. Por eso `env_if_unset()` en `hyde/env.lua` puede recibir valores ya bloqueados.
- Servicios de `startup.lua` usan systemd scopes con `Wants=uwsm-mainston.service` implГӯcito para orden de arranque.
- `dbus-update-activation-environment` en startup.lua es parcialmente redundante bajo UWSM (pero inofensivo).

### 6.2 Secuencia de arranque

```
1. UWSM arranca Hyprland (systemd user unit)
2. Hyprland carga hyprland.lua
3. Lua carga mГіdulos base (variables, env, hardware, etc.)
4. Lua carga estilo desde ~/.local/state/hypr/ (autogenerado)
5. startup.lua ejecuta servicios systemd:
   - waybar, dunst, hypridle, hyprsunset, clipboard, wallpaper
   - polkit, nm-applet, blueman, udiskie, batterynotify
6. hypr.scripts.wallbash_metadata_loader genera metadata.conf
7. Al cambiar wallpaper/theme:
   a. wallbash extrae colores del wallpaper
   b. Genera paleta 4Г—(1+1+9) = 44 colores
   c. Rellena plantillas .dcol/.Xcol/.t2 вҶ’ archivos finales
   d. Scripts post-procesadores (dunst.sh, qtct.sh, reload_kitty.sh) recargan apps
   e. hyprctl reload aplica cambios en Hyprland
8. staterc persiste: HYDE_THEME, HYPR_ANIMATION, HYPR_WORKFLOW, etc.
```

## 9. ConvenciГіn de Nombres de Plantillas

- **`.dcol`** = plantilla de color wallbash вҶ’ lГӯnea 1 = ruta destino, resto = contenido con `<wallbash_*>`
- **`.Xcol`** = variante pywal-compat вҶ’ lГӯnea 1 = ruta destino, formato 16 colores
- **`.t2`** = plantilla binaria/estructurada (SVG, etc.) вҶ’ lГӯnea 1 = ruta destino
- Si lГӯnea 1 tiene `| script.sh` вҶ’ se ejecuta el script post-generaciГіn (ej: `| dunst.sh`, `| qtct.sh`, `| reload_kitty.sh`)
